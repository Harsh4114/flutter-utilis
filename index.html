Flutter Community Utils

A community-driven GitHub repository that contains reusable Flutter utilities, services, helpers, and common implementations to help Flutter developers build applications faster.

🚀 About This Repository

This repository is created for the Flutter community. Developers can contribute reusable modules, utilities, services, and helper classes that can be used across multiple Flutter projects.


Instead of rewriting the same code in every project, developers can find ready-to-use implementations here.

📦 Current Contributions

  • API Dio Client
    Complete Dio setup for sending HTTP requests, interceptors, error handling, and API services.

  • SVG Utility
    Centralized SVG icon management for Flutter applications. Developers can access SVG assets using reusable classes.

🤝 How To Contribute

  1. Fork the repository.
  2. Create a new branch.
  3. Add your Flutter utility or service.
  4. Commit your changes.
  5. Create a Pull Request.

🌿 Branch Naming Convention

Use meaningful branch names:


  • feature/dio-client
  • feature/svg-utils
  • feature/firebase-service
  • feature/cache-manager
  • fix/api-error-handling

📝 Pull Request Guidelines

Every Pull Request should include:


  • Feature Name
  • Folder Name
  • Purpose of the Utility
  • How to Use It
  • Required Dependencies
  • Example Implementation

Example:

Feature: Dio API Client

Folder: api/

Purpose: Send HTTP requests using Dio

⚡ Contribution Rules

  • Code must be reusable.
  • Code should follow Flutter best practices.
  • Provide proper documentation.
  • Keep folder structure clean.
  • Add usage examples whenever possible.
  • Avoid project-specific implementations.
